# دليل تنظيف وإصلاح المواقع الجغرافية 🌍

## 📋 المشكلة

عند معالجة الأخبار، قد يحدث الخلط بين أسماء مشابهة في دول مختلفة:
- **مثال:** "لبنان" (الدولة) يتم الخلط بينها و "بنان" (مدينة في تونس)
- **النتيجة:** إحداثيات خاطئة (خط عرض وطول غلط)

## 🚀 الخطوات الكاملة

### للمرة الأولى:

```bash
# 1. تنظيف البيانات الموجودة
python scripts/cleanup_locations.py

# 2. إعادة معالجة الأخبار
python scripts/reprocess_news.py

# 3. التحقق من النتائج
# استخدم API للتحقق من الأحداث الجديدة
curl "http://localhost:7235/api/v1/news-events?country_code=LB&limit=10"
```

### للمعالجة المستقبلية:

- النظام سيستخدم القائمة البيضاء تلقائياً
- لن يحدث الخلط بين الأسماء المشابهة
- الإحداثيات ستكون صحيحة دائماً

## 🔍 التحقق من النتائج

### 1. التحقق من المواقع:

```sql
-- عرض جميع مواقع لبنان
SELECT * FROM locations 
WHERE country_code = 'LB' 
ORDER BY name;

-- التحقق من الإحداثيات
SELECT name, country_code, latitude, longitude 
FROM locations 
WHERE name IN ('لبنان', 'بيروت', 'دمشق', 'بغداد')
ORDER BY country_code;
```

### 2. التحقق من الأحداث:

```sql
-- عدد الأحداث لكل دولة
SELECT l.country_code, COUNT(*) as event_count
FROM news_events ne
JOIN locations l ON ne.location_id = l.id
GROUP BY l.country_code
ORDER BY event_count DESC;

-- الأحداث المرتبطة بلبنان
SELECT ne.id, ne.place_name, rn.title_original
FROM news_events ne
JOIN locations l ON ne.location_id = l.id
JOIN raw_news rn ON ne.raw_news_id = rn.id
WHERE l.country_code = 'LB'
LIMIT 10;
```

### 3. التحقق من API:

```bash
# الأحداث في لبنان
curl "http://localhost:7235/api/v1/news-events?country_code=LB"

# الأحداث مع مقاييس
curl "http://localhost:7235/api/v1/news-events?has_metrics=true"

# الأحداث في دول محددة
curl "http://localhost:7235/api/v1/news-events?country_code=SY,IQ,PS"
```

## 📝 ملاحظات مهمة

1. **القائمة البيضاء أولاً:** النظام يتحقق من القائمة البيضاء قبل استخدام Nominatim
