## نظرة عامة

النظام يعمل على معالجة الأخبار العربية بشكل دوري (كل 15 دقيقة) لاستخراج:
- **الأماكن الجغرافية** (الدول والمدن)
- **المقاييس الرقمية** (الضحايا، الصواريخ، إلخ)

ثم يربط هذه البيانات معاً في قاعدة البيانات.

---

## كيفية عمل نظام الأحداث

### 1️⃣ نقطة البداية: Scheduler (المجدول)

```
┌─────────────────────────────────────────────────────────────┐
│                    SchedulerManager                         │
│                                                             │
│  يعمل في الخلفية (Background Task)                         │
│  ينفذ مهام دورية بجدول زمني محدد                           │
└─────────────────────────────────────────────────────────────┘
                            │
                            ├─ كل 15 دقيقة: معالجة البيانات
                            │  (استخراج الأماكن والمقاييس)
                            │
                            └─ كل 10 ساعات: توليد التنبؤات
                               (AI Forecast)
```

### 2️⃣ دورة معالجة البيانات (Data Processing Cycle)

```
START
  │
  ├─→ [1] استخراج الأماكن (Extract Locations)
  │        └─ قراءة أخبار عربية
  │        └─ استخراج أسماء الدول والمدن
  │        └─ تحويل الأسماء إلى إحداثيات
  │        └─ حفظ في قاعدة البيانات
  │
  ├─→ [2] استخراج المقاييس (Extract Metrics)
  │        └─ قراءة الأحداث المعالجة
  │        └─ استخراج الأرقام (ضحايا، صواريخ، إلخ)
  │        └─ حفظ في قاعدة البيانات
  │
  └─→ END
```

### المرحلة 3: استخراج الأماكن (NER - Named Entity Recognition)

**الملف:** `app/services/nlp/ner_camel.py`

يستخدم مكتبة **CAMeL Tools** لاستخراج الكيانات المسماة من النص العربي.

```python
def extract_places_ner(text):
    """
    استخراج أسماء الأماكن من النص العربي
    
    الخطوات:
    1. تقسيم النص إلى جمل
    2. تشغيل NER على كل جملة
    3. استخراج الكيانات من نوع LOC (Location) و GPE (Geopolitical Entity)
    4. تطبيع الأسماء (إزالة التشكيل، توحيد الألف، إلخ)
    5. إزالة التكرارات
    """
```

**مثال:**
```
النص:
"غارات إسرائيلية على غزة والبحرين وبالامارات"

NER Output:
- "غزة" → LOC (Location)
- "البحرين" → GPE (Geopolitical Entity)
- "الامارات" → GPE (Geopolitical Entity)

بعد التطبيع:
- "غزة"
- "البحرين"
- "الامارات"
```

**كيفية عمل NER:**

```
الجملة: "غارات إسرائيلية على غزة"
الكلمات: ["غارات", "إسرائيلية", "على", "غزة"]

NER Labels:
- "غارات" → O (Outside)
- "إسرائيلية" → B-GPE (Beginning of Geopolitical Entity)
- "على" → O (Outside)
- "غزة" → B-LOC (Beginning of Location)

النتيجة:
- "إسرائيلية" → GPE
- "غزة" → LOC
```

### المرحلة 3: استخراج المقاييس بـ Regex

**الملف:** `app/services/extraction/metrics_extractor.py`

يستخدم **Regular Expressions** لاستخراج الأرقام والمقاييس.

**أنواع المقاييس:**

#### 1. الصواريخ والطائرات المسيرة
```python
# الصواريخ المطلقة
"missiles_launched": r"(?:أطلق|أطلقت).{0,50}?(\d+)\s*(?:صاروخ|صواريخ)"

# الصواريخ المعترضة
"missiles_intercepted": r"(?:اعترض|اعترضت|أسقط|أسقطت).{0,50}?(\d+)\s*(?:صاروخ|صواريخ)"

# الطائرات المسيرة المطلقة
"drones_launched": r"(?:اطلق|اطلقت).{0,50}?(\d+)\s*(?:طائرة مسيرة|مسيرة|درون)"

# الطائرات المسيرة المعترضة
"drones_intercepted": r"(?:اعترض|اعترضت|اسقط|اسقطت).{0,50}?(\d+)\s*(?:طائرة مسيرة|مسيرة|درون)"
```

#### 2. الضحايا
```python
# القتلى
"killed": r"(?:مقتل|قتل|استشهاد).{0,30}?(\d+)\s*(?:شخص|شهيد|قتيل|ضحية)?"

# المدنيين القتلى
"civilians_killed": r"(?:مقتل|قتل).{0,30}?(\d+)\s*(?:مدني|مواطن)"

# الجنود القتلى
"soldiers_killed": r"(?:مقتل|قتل).{0,30}?(\d+)\s*(?:جندي|عسكري)"

# الجرحى
"injured": r"(?:اصابة|جرح|مصاب).{0,30}?(\d+)\s*(?:شخص|مصاب|جريح)?"
```

#### 3. العمليات العسكرية
```python
# الغارات الجوية
"airstrikes": r"(?:غارة|غارات|ضربة جوية).{0,50}?(\d+)"

# القصف المدفعي
"artillery_shells": r"(?:أطلق|أطلقت).{0,50}?(\d+)\s*(?:قذيفة|قذائف)\s*(?:مدفعية)?"

# الأهداف المستهدفة
"targets_hit": r"(?:استهداف|قصف).{0,50}?(\d+)\s*(?:هدف|موقع)"

# المباني المدمرة
"buildings_destroyed": r"(?:تدمير|دمر|هدم).{0,50}?(\d+)\s*(?:مبنى|منزل|بناية)"
```

**مثال:**
```
النص:
"أطلقت إسرائيل 3 صواريخ على غزة، مما أسفر عن مقتل 15 شخص وإصابة 30 آخرين"

Regex Matches:
1. missiles_launched: 3
2. killed: 15
3. injured: 30

النتيجة:
[
  {"metric_type": "missiles_launched", "value": 3, "snippet": "أطلقت إسرائيل 3 صواريخ..."},
  {"metric_type": "killed", "value": 15, "snippet": "مما أسفر عن مقتل 15 شخص..."},
  {"metric_type": "injured", "value": 30, "snippet": "وإصابة 30 آخرين"}
]
```

## أمثلة عملية

### مثال 1: خبر بسيط

**الخبر الأصلي:**
```
"غارات إسرائيلية على غزة أسفرت عن مقتل 10 أشخاص"
```

**الخطوات:**

1. **استخراج الأماكن:**
   - NER يستخرج: "إسرائيلية" (GPE), "غزة" (LOC)
   - Geocoding:
     - "إسرائيلية" → Israel (IL)
     - "غزة" → Palestine (PS)

2. **استخراج المقاييس:**
   - Regex يستخرج: "10" من "مقتل 10 أشخاص"
   - metric_type: "killed", value: 10

3. **النتيجة في قاعدة البيانات:**
   ```
   news_events:
   - event_id: 1, location_id: 1 (غزة), place_name: "غزة"
   
   event_metrics:
   - event_id: 1, metric_type: "killed", value: 10
   ```

### مثال 2: خبر معقد

**الخبر الأصلي:**
```
"أطلقت إسرائيل ثلاثة صواريخ وطائرتين مسيرتين على غزة والبحرين، 
مما أسفر عن مقتل 15 شخص وإصابة 30 آخرين. 
تم اعتراض صاروخين من قبل الدفاع الجوي."
```

**الخطوات:**

1. **تحويل الأرقام العربية:**
   ```
   "ثلاثة صواريخ" → "3 صواريخ"
   "طائرتين مسيرتين" → "2 طائرة مسيرة"
   "صاروخين" → "2 صاروخ"
   ```

2. **استخراج الأماكن:**
   - "غزة" → Palestine (PS)
   - "البحرين" → Bahrain (BH)

3. **استخراج المقاييس:**
   - missiles_launched: 3
   - drones_launched: 2
   - killed: 15
   - injured: 30
   - missiles_intercepted: 2

4. **النتيجة:**
   ```
   news_events:
   - event_id: 1, location_id: 1 (غزة)
   - event_id: 2, location_id: 2 (البحرين)
   
   event_metrics:
   - event_id: 1, metric_type: "missiles_launched", value: 3
   - event_id: 1, metric_type: "drones_launched", value: 2
   - event_id: 1, metric_type: "killed", value: 15
   - event_id: 1, metric_type: "injured", value: 30
   - event_id: 1, metric_type: "missiles_intercepted", value: 2
   ```

### مثال 3: خبر بحروف جر ملتصقة

**الخبر الأصلي:**
```
"غارات على الامارات والبحرين وبالسعودية"
```

**الخطوات:**

1. **تحضير النص:**
   ```
   "الامارات والبحرين وبالسعودية"
   ↓
   "الامارات و البحرين و ب السعودية"
   ```

2. **استخراج الأماكن:**
   - NER يستخرج: "الامارات", "البحرين", "السعودية"

3. **Geocoding:**
   - "الامارات" → UAE (AE)
   - "البحرين" → Bahrain (BH)
   - "السعودية" → Saudi Arabia (SA)

4. **النتيجة:**
   ```
   news_events:
   - event_id: 1, location_id: 1 (الامارات)
   - event_id: 2, location_id: 2 (البحرين)
   - event_id: 3, location_id: 3 (السعودية)
   ```

## ملاحظات مهمة

1. **معالجة الأخطاء:** إذا فشل استخراج الأماكن، يتم تخطي الخبر والمتابعة مع الخبر التالي.

2. **التخزين المؤقت:** الأماكن المستخرجة تُخزن في جدول `locations` لتجنب إعادة المعالجة.

3. **معدل الطلبات:** Nominatim لديه حد أقصى للطلبات (1 طلب/ثانية)، لذلك يتم معالجة الأماكن بشكل تدريجي.

4. **الدقة:** NER قد لا يستخرج جميع الأماكن بدقة 100%، خاصة الأماكن الصغيرة أو الأسماء الغير شهيرة.

5. **المقاييس:** قد تكون هناك أرقام كاذبة (false positives)، لذلك يتم تصفيتها بناءً على نطاق معقول (0 < value < 100,000).
