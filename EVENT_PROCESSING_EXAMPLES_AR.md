## مثال 2: خبر معقد مع أرقام عربية

### الخبر الأصلي:
```
"أطلقت إسرائيل ثلاثة صواريخ وطائرتين مسيرتين على غزة والبحرين، 
مما أسفر عن استشهاد خمسة عشر شخص وإصابة ثلاثين آخرين. 
تم اعتراض صاروخين من قبل الدفاع الجوي."
```

### المعالجة:

#### 1. تحويل الأرقام العربية
```
"ثلاثة صواريخ" → "3 صواريخ"
"طائرتين مسيرتين" → "2 طائرة مسيرة"
"خمسة عشر شخص" → "15 شخص"
"ثلاثين آخرين" → "30 آخرين"
"صاروخين" → "2 صاروخ"

النص بعد التحويل:
"أطلقت إسرائيل 3 صواريخ و 2 طائرة مسيرة على غزة و البحرين، 
مما أسفر عن استشهاد 15 شخص وإصابة 30 آخرين. 
تم اعتراض 2 صاروخ من قبل الدفاع الجوي."
```

#### 2. استخراج الأماكن
```
NER Output:
- "إسرائيلية" → GPE
- "غزة" → LOC
- "البحرين" → LOC

Geocoding:
- "إسرائيلية" → Israel (IL), lat: 31.95, lng: 35.20
- "غزة" → Palestine (PS), lat: 31.95, lng: 35.20
- "البحرين" → Bahrain (BH), lat: 26.07, lng: 50.56
```

#### 3. استخراج المقاييس
```
Regex Matches:
1. missiles_launched: 3
2. drones_launched: 2
3. killed: 15
4. injured: 30
5. missiles_intercepted: 2

metrics = [
  {"metric_type": "missiles_launched", "value": 3},
  {"metric_type": "drones_launched", "value": 2},
  {"metric_type": "killed", "value": 15},
  {"metric_type": "injured", "value": 30},
  {"metric_type": "missiles_intercepted", "value": 2}
]
```

#### 4. النتيجة في قاعدة البيانات
```
news_events:
- event_id: 1, raw_news_id: 101, location_id: 1, place_name: "إسرائيلية"
- event_id: 2, raw_news_id: 101, location_id: 2, place_name: "غزة"
- event_id: 3, raw_news_id: 101, location_id: 3, place_name: "البحرين"

event_metrics:
- event_id: 1, metric_type: "missiles_launched", value: 3
- event_id: 1, metric_type: "drones_launched", value: 2
- event_id: 1, metric_type: "killed", value: 15
- event_id: 1, metric_type: "injured", value: 30
- event_id: 1, metric_type: "missiles_intercepted", value: 2
```

## مثال 6: خبر بأرقام كاذبة (False Positives)

### الخبر الأصلي:
```
"في الساعة 15:30 من يوم الثلاثاء، وقع حادث في الطريق السريع رقم 5"
```

### المعالجة:

#### 1. استخراج الأماكن
```
NER Output:
- "الطريق السريع" قد لا يتم استخراجه (ليس مكان محدد)
```

#### 2. استخراج المقاييس
```
Regex Matches:
- "15" من "15:30" (ساعة)
- "5" من "رقم 5" (رقم طريق)

لكن هذه أرقام كاذبة (false positives)
```

#### 3. التصفية
```
# يتم تصفية الأرقام غير المعقولة
if value <= 0 or value > 100000:
    continue

# في هذه الحالة:
- 15 و 5 قد تكون ضمن النطاق المعقول
- لكن السياق يشير إلى أنها ليست مقاييس حقيقية

ملاحظة: هذا يعتمد على جودة النمط (regex)
```

#### 4. النتيجة
```
event_metrics:
(قد تكون فارغة أو تحتوي على أرقام كاذبة)
```

## ملخص الحالات الخاصة

| الحالة | المثال | النتيجة |
|--------|--------|--------|
| خبر بسيط | "غارات على غزة" | 1 مكان، 0 مقاييس |
| أرقام عربية | "ثلاثة صواريخ" | تحويل إلى 3 |
| حروف جر ملتصقة | "والبحرين" | فصل إلى "و البحرين" |
| أسماء مدن صغيرة | "رفح وخان يونس" | استخراج كلا المدينتين |
| أسماء غير شهيرة | "قرية صغيرة" | قد لا يتم استخراجها |
| أرقام كاذبة | "الساعة 15:30" | قد يتم استخراج 15 و 30 |
| أسماء متشابهة | "دمشق" مرتين | إزالة التكرارات |
| لهجات مختلفة | "بيروت والعاصمة اللبنانية" | استخراج كلا الاسمين |
| أسماء وهمية | "مدينة خيالية" | لا يتم استخراجها |
| أحرف مختلفة | "إيران وايران" | توحيد إلى "ايران" |
| أرقام كبيرة جداً | "مليون شخص" | تصفية (> 100000) |
| أرقام صفر | "لم يتم قتل أي شخص" | تصفية (<= 0) |
| أسماء متعددة الكلمات | "نيويورك وسان فرانسيسكو" | استخراج كلا الاسمين |
