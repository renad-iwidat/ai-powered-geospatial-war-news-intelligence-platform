## 🌍 استخراج الأماكن (6 خطوات)

### الخطوة 1: قراءة الأخبار العربية
```python
# من قاعدة البيانات
SELECT raw_news_id, title_ar, content_ar
FROM raw_news
WHERE language_id = 1 (عربي)
AND NOT EXISTS (news_events)  # لم تتم معالجتها
```

### الخطوة 2: تحضير النص
```python
# فصل حروف الجر عن أسماء الأماكن
"والبحرين" → "و البحرين"
"بالسعودية" → "ب السعودية"
```

### الخطوة 3: استخراج الأماكن (NER)
```python
# استخدام CAMeL Tools
NER يستخرج:
- LOC: Location (موقع)
- GPE: Geopolitical Entity (كيان جيوسياسي)

مثال:
"غارات على غزة والبحرين"
→ ["غزة", "البحرين"]
```

### الخطوة 4: تحويل إلى إحداثيات (Geocoding)
```python
# استخدام Nominatim (OpenStreetMap)
"البحرين" → {
  "name": "Bahrain",
  "country_code": "BH",
  "lat": 26.0667,
  "lng": 50.5577
}
```

### الخطوة 5: حفظ الأماكن
```python
INSERT INTO locations
(name, country_code, latitude, longitude, osm_id, osm_type)
VALUES ('البحرين', 'BH', 26.0667, 50.5577, 305938, 'relation')
```

### الخطوة 6: ربط الأخبار بالأماكن
```python
INSERT INTO news_events
(raw_news_id, location_id, place_name, event_type)
VALUES (123, 2, 'البحرين', 'unknown')
```

## 📈 استخراج المقاييس (4 خطوات)

### الخطوة 1: تحويل الأرقام العربية
```python
# تحويل الأرقام الكتابية إلى أرقام
"ثلاثة صواريخ" → "3 صواريخ"
"طائرتين مسيرتين" → "2 طائرة مسيرة"

# قاموس الأرقام:
واحد=1, اثنين=2, ثلاثة=3, ..., ألف=1000
```

### الخطوة 2: استخراج بـ Regex
```python
# أنماط للمقاييس المختلفة
missiles_launched: r"(?:أطلق|أطلقت).{0,50}?(\d+)\s*(?:صاروخ|صواريخ)"
killed: r"(?:مقتل|قتل).{0,30}?(\d+)\s*(?:شخص|شهيد|قتيل)?"
injured: r"(?:اصابة|جرح).{0,30}?(\d+)\s*(?:شخص|مصاب|جريح)?"
```

### الخطوة 3: التصفية
```python
# تصفية الأرقام غير المعقولة
if value <= 0 or value > 100000:
    continue
```

### الخطوة 4: حفظ المقاييس
```python
INSERT INTO event_metrics
(event_id, metric_type, value, snippet)
VALUES (1, 'missiles_launched', 3, 'أطلقت إسرائيل 3 صواريخ...')
```

## 🔧 أنواع المقاييس المدعومة

- `missiles_intercepted`: الصواريخ المعترضة

- `tanks_deployed`: الدبابات المنتشرة

## 🚀 الأداء والتحسينات

### الأداء الحالي
- معالجة 50 خبر كل 15 دقيقة (استخراج الأماكن)
- معالجة 100 حدث كل 15 دقيقة (استخراج المقاييس)
- معدل Nominatim: 1 طلب/ثانية

### التحسينات الممكنة
1. **استخدام نموذج NER أفضل** (مثل BERT)
2. **تحسين نماذج Regex** (إضافة حالات جديدة)
3. **استخدام خدمة geocoding أفضل** (Google Maps)
4. **إضافة معالجة للأسماء البديلة** (synonyms)
5. **تحسين معالجة الأخطاء** (retry logic)

## 📝 ملاحظات مهمة

### 1. معالجة الأخطاء
- إذا فشل استخراج الأماكن: يتم تخطي الخبر
- إذا فشل Geocoding: يتم تخطي المكان
- إذا فشل استخراج المقاييس: يتم تخطي الحدث

### 2. التخزين المؤقت
- الأماكن المستخرجة تُخزن لتجنب إعادة المعالجة
- البحث عن مكان موجود قبل إدراج جديد

### 3. منع التكرار
- استخدام `ON CONFLICT` في SQL
- إزالة التكرارات من نتائج NER

### 4. معدل الطلبات
- Nominatim: 1 طلب/ثانية
- معالجة أول 3 أماكن فقط لكل خبر

### 5. الدقة
- NER قد لا يستخرج جميع الأماكن بدقة 100%
- Regex قد تستخرج أرقام كاذبة (false positives)
- التصفية تساعد في تقليل الأخطاء

## 📞 الدعم والمساعدة

### الأسئلة الشائعة

**س: لماذا لم يتم استخراج مكان معين؟**
- قد يكون NER لم يستخرجه
- قد يكون Nominatim لم يجده
- قد يكون اسم المكان غير شهير

**س: لماذا تم استخراج رقم كاذب؟**
- قد يكون الرقم في سياق مختلف (مثل ساعة)
- قد يكون نمط Regex غير دقيق

**س: كم مرة يتم معالجة الأخبار؟**
- كل 15 دقيقة للأماكن والمقاييس
- كل 10 ساعات للتنبؤات

**س: هل يمكن تغيير معدل المعالجة؟**
- نعم، في `scheduler.py`:
  ```python
  trigger=IntervalTrigger(minutes=15)  # غيّر الرقم
  ```
